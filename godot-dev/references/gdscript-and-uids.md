# GDScript 与 UID

- `.gd` 直接编辑磁盘；保持静态类型、现有代码风格和项目生命周期约束。
- `class_name` 会影响全局脚本类缓存。新增、删除、改名或改变基类后执行编辑器扫描，必要时完整重启。
- `.gd.uid` 由 Godot 管理。移动脚本时连同 UID 文件一起移动；不要为新脚本伪造 UID。
- 场景和资源中的 `uid://` 必须能由目标项目解析。手工创建新引用时可先使用 `res://` 路径，交由 Godot 导入和保存补齐 UID。
- 单文件语法检查不能完整覆盖全局类、AutoLoad 和插件环境；最终必须在项目上下文执行 Headless 编辑器扫描。
- `@tool` 脚本会在编辑器运行。修改前检查副作用，修改后检查编辑器 Output、Debugger 和插件状态。
